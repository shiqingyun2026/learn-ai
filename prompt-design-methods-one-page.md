# Prompt 设计一页课件版

## 1. Prompt 设计的核心

`Prompt 不是一句提问，而是任务、规则、输出和边界的组合设计。`

做产品时，Prompt 设计的目标不是“写得像高手”，而是让模型：
- 做对事
- 少乱答
- 输出稳
- 能接流程

---

## 2. 四类 Prompt 及目标

### 分类类 Prompt

目标：
- 回答“这是什么问题”
- 把输入稳定映射到封闭标签集合

设计重点：
- 标签闭集
- 标签边界
- 优先级规则
- 固定输出

---

### 建议类 Prompt

目标：
- 回答“接下来怎么处理”
- 输出具体、可执行的动作建议

设计重点：
- 建议可执行
- 信息不足先核验
- 高风险先升级
- 输出结构固定

---

### 检索问答类 Prompt

目标：
- 回答“当前资料到底怎么规定”
- 仅基于检索到的资料作答

设计重点：
- 只基于资料回答
- 资料不足就承认不足
- 资料冲突就承认冲突
- 不用常识脑补企业规则

---

### Wiki 类 Prompt

目标：
- 回答“通常怎么理解、怎么总结”
- 基于整理后的知识页进行汇总和解释

设计重点：
- 强调总结和归纳
- 守住 wiki 资料边界
- 涉及精确条款时提示回查原文

---

## 3. Prompt 最佳设计流程

`先定任务，再定输出，再写 Prompt，再用 badcase 迭代。`

标准流程：
1. 定义任务目标
2. 定义输出合同
3. 定义评测标准
4. 准备样本
5. 写基础版 Prompt
6. 跑测试
7. 找错误模式
8. 按错误补规则
9. 必要时补 few-shot
10. 冻结版本并持续收 badcase

---

## 4. Prompt 规则怎么写：5 类规则

推荐顺序：

`先收住输出 -> 防止乱编 -> 提高质量 -> 写清业务规则 -> 固定交付格式`

### 1. 输出闭集约束
- 允许输出哪些标签、字段值、枚举值
- 防止模型自造标签

示例：
- `只能输出一个标签`
- `primary_issue 只能是以下 6 类`

### 2. 防止乱编约束
- 信息不足怎么办
- 资料缺失怎么办
- 哪些内容不能推断

示例：
- `如果资料不足，直接输出“当前资料不足”`
- `不要补充资料外规则`

### 3. 建议质量约束
- 建议是否具体
- 是否可执行
- 是否面向下一步动作

示例：
- `建议必须具体、可执行`
- `不要输出“加强、优化、持续关注”`

### 4. 业务判定规则
- 多问题怎么判主问题
- 冲突场景按什么优先级处理
- 高风险如何分流

示例：
- `后半句明确投诉时，优先按投诉内容分类`
- `高风险问题优先回访/核验/升级`

### 5. 输出格式约束
- 最终输出长什么样
- 是否必须 JSON
- 是否允许额外解释

示例：
- `只输出合法 JSON`
- `不要输出 markdown 或额外说明`

---

## 5. 一句话记忆

- 分类类：`先收住标签`
- 建议类：`先收住动作`
- 检索问答类：`先守住证据边界`
- Wiki 类：`先守住知识层边界`

总口诀：

`Prompt 优化不是润色文案，而是围绕错误模式持续补规则。`
